Theoretical Background

In multi-dimensional data analysis, users often face the difficulty of making choices based on conflicting criteria. Skyline queries (also known as Pareto optimization) extract the set of optimal solutions by filtering out data points that are "dominated" by other points.

However, as dimensions increase, the Skyline set often remains large and difficult to interpret. This project re-implements and extends the SkyLens system proposed by Zhao et al. (2018). Through Coordinated Multiple Views, the system not only displays the Skyline set but also intuitively explains why a point belongs to the Skyline and in which subspaces it excels by quantifying the "Dominance Score".

Implementation Details

This project is not just a simple re-implementation but also includes performance optimizations for modern browser environments. The core architecture adopts a component-based design using React 19, builds with Vite, and ensures code robustness with TypeScript.

  • WebGPU Acceleration: To address the O(n²) computational complexity of the original algorithm, we implemented a parallel computation module based on WebGPU. By processing Dominance comparisons in parallel using Compute Shaders, smooth interactive experiences are achieved even with large datasets (an improvement not covered in the original paper).
  • Custom Visualizations (D3.js): All view components (ProjectionView, RadarChart, TabularView) are custom-developed using D3.js rather than relying on off-the-shelf charting libraries. For instance, the Glyphs in the ComparisonView are dynamically generated via SVG paths, capable of simultaneously encoding attribute values and dominance relationships.
  • Interactive Linking: Global state management is implemented. Data filtered by the Brush in the Tabular View updates the t-SNE layout in the Projection View in real-time; mouse hover operations trigger cross-highlighting across all views.

Application Features

The system re-implements the three core views from the paper and maintains high interactivity:

  • Projection View: Uses dimensionality reduction algorithms to map high-dimensional Skyline points onto a 2D plane. The color of the points encodes their Dominance Score, while the outer ring charts display the numerical distribution of key attributes.
  • Tabular View: Utilizes "Diverging Bar Charts" to display data details. Each row represents a data point, with blue/red bars intuitively reflecting the point's performance relative to the overall average.
  • Comparison View: A view dedicated to in-depth comparison. After users select points of interest, the system generates composite Glyphs of radar charts and pie charts, intuitively showcasing the Dominance Strength and differences between two points.
